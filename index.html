Scope

Minimal, extensible web-based Project Tracking service for SciLifeLab units with consistent project records, role-based views (users, staff, management), simple lifecycle and milestones, and notifications.

Out of scope

Full User Portal functionality; replacement of ordering, billing or LIMS systems, and complex workflow automation beyond basic statuses and milestones.

Value

  • Give users a clear overview of their projects (status, next steps, contacts) and reduce email back-and-forth.
  • Give staff and unit leadership a shared view of project portfolios, workloads and bottlenecks for planning and reporting.
  • Provide SciLifeLab with comparable cross-platform project data and evidence to inform future User Portal and IDS developments.

Main deliverables

  • Project Tracking service for the SciLifeLab community (staff and researchers), providing clear project overviews, statuses, next steps and contacts.
  • Minimal, robust architecture, data model and integration design that enables the service to interoperate with IAM, Order Portal (or similar) and identifier services.
  • Production-ready web application adopted by an initial 2–4 units and, by project end, a larger set of additional units beyond the pilot group.

Dependencies

  • Identity and Access Management / LifeScience ID and role models (including privacy, security and audit).
  • Sample and project identifier services (e.g. BioSamples/EMBL-EBI instance, RAiD).
  • Order Portal and relevant unit systems where interoperability adds value.
  • Overall IDS strategy for cross-platform workflows and future automation.

High-level architecture

Simplified view of the project tracking service, integrating existing unit systems (Order Portal, iLAB, FMS, others) into a central project database and web interface.
